# Hero slider — CollectionStage

**Problem.** Левая колонка hero (sticky aside, ~440px) должна нести
«top collection» без скучной плитки. Первый прототип: круглая
аватарка бренда + крупное название + пара стрелок, CSS-transitions
на смене. Ощущалось как слайд-шоу из 2014, а не как premium hero.

**Tried.**

1. **Фон-варианты.** Крупное фото часов на всю высоту aside. Это —
   the bet. Других форматов для этого места не пробовали, потому что
   «окно в выставку» идёт слева, и часы должны дублировать приём
   живой визуальной точки входа.

2. **Transition: CSS keyframes vs framer-motion.** Сначала попробовал
   custom keyframes `slider-in-*/slider-out-*` в `index.css`. Быстро
   упёрлись в два огр-ения: (a) одновременно старое+новое фото в DOM
   → CSS-анимации не знают друг о друге, нет chaining; (b) direction-
   aware (prev vs next) требует либо class-juggling, либо cleanup
   на animationend. Перешли на framer-motion + `AnimatePresence`.
   custom prop для direction, variants `enter/center/exit` — чисто
   и декларативно. ✓ **selected — framer-motion.**

3. **Spring vs easing для x.** Spring (stiffness 240, damping 30)
   добавлял еле-слышный оверштут — на таких длинных durations (4s)
   он начинал «звучать» и конфликтовал с opacity/scale на чистом
   easing. Убрал spring, всё на одной кривой cubic-bezier
   `[0.22, 1, 0.36, 1]` (ease-out quart). Старт быстрый, хвост
   длинный — ощущение «отпущенного» движения. ✓

4. **mode="wait" vs crossfade.** mode="wait" → старое фото полностью
   exits, потом новое entrers. Никакого наслоения, но возникает
   «пропасть» ~1.5–2s, где экран пуст. crossfade (без mode="wait")
   → смазывается в середине, два призрака одновременно. Компромисс:
   crossfade + `delay: 1.25s` на enter. Старое фото успевает отойти
   и притухнуть до opacity ~0.3, только тогда новое начинает
   въезжать из противоположной стороны. ✓

5. **Сдвиг по x.** Прошли через 64 → 80 → 520 → 220 → 100px. 520
   создавало «жёсткий обрыв» об край контейнера; 80 — слишком
   скромно, не ощущалось как движение. **100px** — лёгкий, но
   читаемый сдвиг. overflow-hidden на контейнере снят, чтобы не
   возникало визуального «клипа» у края.

6. **Длительности.** Исходно 0.5s по x + spring. Растянули до 4s
   по x + 3.2s по opacity + 4s по scale, чтобы транзиция ощущалась
   как «вдох-выдох», а не «щёлк-щёлк». Info-блок (brand eyebrow +
   collection name) идентично фото — 4s/3.2s + delay 1.25.

7. **Arrow-направления.** Инвертировали: ← (клавиша/кнопка) — часы
   едут справа налево (motion leftward, `next()`); → — часы едут
   слева направо (motion rightward, `prev()`). Логика «куда стрелка
   — туда движение», а не «prev/next» в абстрактной нумерации.

8. **Первый тик.** Auto-rotate 9s слишком долгий для первой
   показательной ротации — пользователь 7–9s не понимает, что это
   слайдер. `firstRun` ref: первый тик через 2500ms, дальше — 9s.

**Decision.** Сейчас в проде: 3 слайда (Tudor Royal / Bulgari
Serpenti / IWC Pilot), 960×960 webp с alpha, без drop-shadow (своя
«студийная» тень встроена в pixels). framer-motion + AnimatePresence
без `mode="wait"`, delay 1.25s на enter. Easing `[0.22, 1, 0.36, 1]`
по всему. x ±100px, opacity 1→0 crossfade, scale 0.97→1. Click
по фото-контейнеру = next. Клавиатурные ←/→ инвертированы
относительно prev/next по направлению движения. Auto-rotate:
первый тик 2.5s, дальше 9s.

**Notes.**
- `drop-shadow-[0_60px_70px_rgba(0,0,0,0.65)]` пришлось убрать —
  на webp с неровной альфой он рендерил квадратную тень от
  bounding-box'а альфы, а не от силуэта часов. Тени теперь
  встроены в сами pixels изображения.
- Info-блок (название коллекции) обёрнут в `grid-cols-[1fr]` с
  `gridArea: '1 / 1'` на обоих motion.div — иначе при отсутствии
  `mode="wait"` оба мотируются в flex-контейнер одновременно,
  ширина пляшет, стрелки справа дёргаются. CSS-grid-one-cell
  стекает слои без влияния на layout sibling'ов — тот же
  паттерн, что и для фото.
